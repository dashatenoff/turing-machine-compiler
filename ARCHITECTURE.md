# Архитектура двухленточной машины Тьюринга

## Лента 1

Содержит входные данные.

Пример:

101+110

Во время вычислений может читаться и при необходимости изменяться.

## Лента 2

Рабочая память и область результата.

Начальное содержимое:

---

После завершения вычислений содержит результат.

Пример:

1011

## Пустой символ

Используется символ:

_

Он обозначает:

* пустую ячейку
* конец данных
* незаполненную память

## Начальное положение головок

Head1 = 0

Head2 = 0

То есть обе головки стоят на первом символе своих лент.

## Завершение программы

Состояние HALT считается завершающим.

## Calling Convention макросов

На входе любого макроса:

* Head1 находится на первом символе данных
* Head2 находится на первом символе рабочей области

На выходе любого макроса:

* Head1 возвращается к началу данных
* Head2 возвращается к началу результата

## Представление сложения

Вход:

101+110

Результат:

1011

Результат располагается на второй ленте.

## Представление инверсии

Вход:

1011

Результат:

0100

Результат располагается на второй ленте.

Reset:
- очищает Tape1;
- очищает Tape2;
- устанавливает текущее состояние в startState;
- обнуляет счётчик шагов.


## Формат состояний

Состояния являются строками.

Примеры:

q0

q1

copy

add

HALT

Состояние HALT зарезервировано и используется для завершения выполнения программы.

---

## Поиск правил

Для выполнения шага машина ищет правило, для которого одновременно выполняются условия:

currentState == rule.currentState

readTape1 == rule.readTape1

readTape2 == rule.readTape2

Если правило найдено:

* выполняется запись символов;
* выполняется перемещение головок;
* происходит переход в следующее состояние.

Если правило не найдено:

* машина считается остановленной;
* step() возвращает false.

---

## Счётчик шагов

После каждого успешного применения правила:

steps++

Счётчик используется GUI для отображения количества выполненных шагов.

---

## Отрицательные индексы

Обе ленты допускают отрицательные индексы.

Пример:

-2 -> '1'
-1 -> '0'
0 -> '1'

Лента считается бесконечной в обе стороны.

---

## Правила для Compiler

Compiler не работает напрямую с Tape и TuringMachine.

Compiler генерирует только:

TMProgram
Rule
StateGenerator

Compiler не должен предполагать внутреннее устройство лент.

Взаимодействие с машиной осуществляется исключительно через генерацию правил.

---

## Контракт между Compiler и макросами

Compiler обязан передавать:

entryState
exitState

Макрос обязан:

* начать выполнение из entryState;
* завершить выполнение в exitState;
* не использовать имена состояний, созданные вне макроса;
* не изменять соглашение о положении головок.

---

## Контракт StateGenerator

Все имена состояний должны быть уникальны.

Примеры:

while_cond_1

while_body_1

while_end_1

invert_scan_5

invert_finish_5

Запрещается вручную создавать состояния с фиксированными именами внутри Compiler.

Все служебные состояния создаются через StateGenerator.

---

## Интерфейс условия есть_бит

Условие:

есть_бит

определяется как:

текущая ячейка Tape1 не содержит символ '_'.

Формально:

tape1.read() != '_'

---

## Что обязан предоставить Parser

Parser должен сообщить и зафиксировать:

* список поддерживаемых команд;
* структуру AST;
* формат цикла пока;
* формат объявления лент;
* текст сообщений об ошибках.

---

## Что обязан предоставить Compiler

Compiler должен зафиксировать:

* какие AST-узлы поддерживаются;
* какие макросы вызываются для каждой команды;
* как организованы циклы пока;
* какие состояния генерируются автоматически.

---

## Интеграционный тест

После завершения основных этапов должны проходить тесты:

invert.mt

Вход:

1011

Результат:

0100

---

add.mt

Вход:

101+110

Результат:

1011

---

После завершения программы:

currentState == HALT

head1 находится в начале данных

head2 находится в начале результата

## Список токенов Lexer

TAPE        — "лента"
WHILE       — "пока"
END         — "конец"
HAS_BIT     — "есть_бит"
CMD_INVERT  — "инвертировать"
CMD_ADD     — "сложить"
COLON       — ":"
STRING      — строка данных, например "101+110"
END_OF_FILE — конец потока

## Комментарии

Строки начинающиеся с # игнорируются Lexer-ом.

## Ошибки Lexer

Неизвестный символ: "Неизвестный символ 'X' на строке N"
Неизвестное слово:  "Неизвестное слово 'X' на строке N"
